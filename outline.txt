研究計画：バブル型クラスタ学習によるGateRAF強化と脳fMRIタスク予測

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
0. なぜGateRAFか
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

タスク実行中の脳fMRI時系列データは以下の3つの性質を持ち，通常の予測モデルでは対処が難しい．

【系列の多さ】
  - fMRIデータはROI×被験者の組み合わせで膨大な数の時系列が存在する
  - 単一被験者・単一ROIのデータ量は限られていても，系列の総数は多い
  - → 他の被験者・ROIの系列を参照することで情報不足を補える
  - RAFはこの「横断的な参照」を可能にするアーキテクチャである

【スパイク】
  - タスク中の神経応答はBOLD信号として現れ，刺激提示タイミングに同期した
    スパイク状の一過性の活動が平常期の中に点在する
  - 予測モデルは誤差全体を最小化しようとするため，頻度の低いスパイクを無視しがち
  - スパイクそのものを予測するよりも，スパイク直前の予兆（トリガー）を
    他被験者の類似タスク事例から参照することで補完できる
  - → RAFの参照機構がスパイク予測に有効

【スパース】
  - タスク中に活動する脳領域は全ROIのごく一部であり，時系列はスパース
  - スパースな系列では時系列モデルが学習できる特徴量が少ない
  - → 似た脳ネットワーク特性を持つ被験者の隠れ状態を参照して補完できる

【シンプルなRAFでは不十分な理由】
  - 似ていない系列を強制的に参照してしまう → ノイズになる
  - 異なる被験者の時系列を結合すると時系列が不連続になる
  - → Gateで不適切な参照を棄却し，Multi-Streamで系列を分離するGateRAFが必要

【脳データ固有の追加課題】
  - fMRI時系列は個人差が激しく，同じタスクでも被験者間で活動パターンが異なる
  - 個人差によってベクトル空間の基準がずれ，参照がさらに機能しにくい
  - → バブル型クラスタ学習で空間を整地することが不可欠

1. 研究の核心
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

対照学習（Contrastive Learning）によるバブル形成で GateRAF が有効に機能する
条件を整え，脳fMRIデータからタスク行動を予測する．

GateRAFが機能するための前提条件：
  - 参照先の隠れ状態が同じベクトル空間の基準を持つこと
  - 似たユーザの隠れ状態が近傍に存在すること

バブルの役割：
  - 個人差・環境差によるベクトル空間のずれを encoder 層で整地
  - 似た脳ネットワーク特性を持つユーザを同一バブルに集約
  - バブル内でのみRAFを実施 → 意味のある参照が可能になる

バブル管理の核心（対照学習版設計）：
  - 全被験者の encoder 出力 h をサーバーが直接観測する
  - 各バブルのプロトタイプ c_B = mean({ h_i | i ∈ B }) を管理する
  - InfoNCE 損失で「同一バブルの h を引き寄せ，異バブルの h を遠ざける」
    表現空間を能動的に整形する
  - → 統計量の間接比較ではなく，損失関数によってバブル構造を直接学習する

2. アーキテクチャ設計：1ラウンドの詳細フロー
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【モデル構成】

  z ∈ R^{288}  (SwiFT 凍結エンコーダの出力)
      ↓
  encoder 層（学習） → h ∈ R^{128}
      ↓
  予測ヘッド（学習） → [acc_2bk_pred, rt_2bk_pred]

  encoder = Linear(288→128) → LayerNorm → ReLU

【1ラウンドの処理フロー】

  ① ラウンド先頭でプロトタイプを凍結（frozen keys）
       h_i^{frozen} = encoder(z_i).mean(windows)    # no_grad，全被験者
       c_B = mean({ h_i^{frozen} | i ∈ B })         # バブルごとの代表点

  ② LOCAL_EPOCHS 回，全被験者を一括で学習
       h_i = encoder(z_i)                            # grad あり（queries）
       pred_i = head(h_i)

       ℒ_task = (1/N) Σ_i MSE(pred_i, labels_i)

       ℒ_con = (1/N) Σ_i InfoNCE(h_i.mean(0), {c_B})
             = (1/N) Σ_i [ -log  exp(sim(h_i, c_{B_i}) / τ)
                                 ─────────────────────────── ]
                                 Σ_{B'} exp(sim(h_i, c_{B'}) / τ)

       ℒ = ℒ_task + λ · ℒ_con  →  backward → Adam

  ③ 評価（no_grad，全被験者）
       各被験者の eval_loss を記録

  ④ バブル管理（スケジュールに従って実行）
       毎 3 ラウンド：Reassign（最近傍プロトタイプへ再割り当て）
       毎 5 ラウンド：Split
       毎 10 ラウンド：Merge

3. バブル更新：コサイン距離に基づく再割り当て・分割・結合
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【距離の定義】

  コサイン距離 d(a, b) = 1 − cos(a, b) ∈ [0, 2]

  h ベクトルを L2 正規化した上でコサイン類似度を計算する．

【再割り当て（Reassign）：3ラウンドごと】

  各被験者 i を最近傍プロトタイプのバブルへ移動する：
    B_i ← argmin_{B} d(h_i, c_B)

  学習で h 空間が更新された結果，別バブルに近づいた被験者を逐次移動させる．

【分割（Split）：5ラウンドごと】

  バブル内の全ペアについてコサイン距離を計算し，
  最大距離が split_threshold（= 0.3）を超えたら2分割する：
    - 最遠ペア (i*, j*) を見つける
    - 各被験者 k を，i* と j* のどちらに近いかで割り振る

【結合（Merge）：10ラウンドごと】

  バブル間プロトタイプのコサイン距離が merge_threshold（= 0.05）以下なら統合する：
    d(c_A, c_B) < 0.05  →  B を A に吸収

【バブル形成の自然な動態】

  初期：全員1バブル
    → ℒ_con = 0（プロトタイプが1つ），ℒ_task だけで h が予測に有用な空間に収束

  Split 後：2バブル以上になると ℒ_con が機能し始める
    → 同一バブルの h が引き寄せられ，異バブルの h が押し広げられる
    → Reassign でバブル帰属が動的に更新される

  収束：分割・結合・再割り当てを繰り返しながら
    → h 空間が「バブルごとにまとまった塊」として安定する

4. 対照学習なしのベースラインとの比較
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ┌────────────────────────┬──────────────────────────┬──────────────────────────────────┐
  │                        │   ベースライン              │   対照学習版（本提案）             │
  ├────────────────────────┼──────────────────────────┼──────────────────────────────────┤
  │ バブル判定の根拠         │  h の分散（受動的観測）    │  InfoNCE 損失（能動的整形）        │
  │ h 空間への介入           │  なし                    │  あり（バブル構造に向けて学習）     │
  │ バブル内の凝集性         │  学習の副産物              │  損失関数が明示的に保証            │
  │ バブル間の分離性         │  学習の副産物              │  損失関数が明示的に保証            │
  │ 損失関数                │  ℒ_task のみ             │  ℒ_task + λ · ℒ_con             │
  │ sweep の対象            │  split/merge 閾値         │  + λ（lambda_con），τ（温度）     │
  └────────────────────────┴──────────────────────────┴──────────────────────────────────┘

  優位点：
  - h 空間がバブル構造を反映するように能動的に学習される
  - split/merge の判定精度が向上（距離の意味が損失と一致）
  - λ = 0 がそのままベースラインになり，消去法的な比較実験が容易

5. バブル形成の3段階（HCPを使った事前学習）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 1：初期バブル形成（HCP WorkingMemoryタスクfMRI）

  目標：被験者の生理的・認知的特性に基づく初期クラスタを形成する

  HCP WMタスクfMRI（生ボクセル，91×109×91×T）
      ↓
  前処理（96³クロップ・z正規化）
      ↓
  事前学習済みエンコーダ SwiFT（凍結） → z ∈ R^{288}
      ↓
  encoder 層（学習） → h ∈ R^{128}
      ↓ ← InfoNCE 損失がバブル構造を整形
  予測ヘッド（学習） → [acc_2bk_pred, rt_2bk_pred]

  サーバーが全被験者の h を直接観測し，プロトタイプを管理する

【バブル形成の3段階】
  1. 初期：全員1バブル，ℒ_task のみで h が予測に有用な空間に収束
  2. 中期：Split後に ℒ_con が機能し始め，h 空間がバブル構造に向けて整形される
  3. 終期：Reassign・Split・Merge を繰り返してバブル構成が安定する

6. 使用データセット
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

HCP (Human Connectome Project) Young Adult
  - 被験者数：1,200名（健常若年成人，22-35歳）
  - タスクfMRI：WorkingMemoryタスク（N-back，LR/RL 2方向）を使用
  - fMRI仕様：2.0mm isotropic，TR=720ms，全脳72スライス，405時点
  - 行動データ：条件別反応時間・正答率が被験者ごとに取得済み
  - アクセス：AWS S3（s3://hcp-openaccess/HCP_1200/）

事前学習済みエンコーダ：SwiFT（NeurIPS 2023）
  - 4D volumetric fMRIを入力とするSwin Transformer
  - HCP・ABCD・UKBiobankで学習済み
  - 入力：(B, 1, 96, 96, 96, 20)，出力：(B, 288, 2, 2, 2, 20) → poolingで (B, 288)
  - 凍結して使用し，encoder 層と予測ヘッドのみ学習

7. 期待される効果
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

・バブルなしのGateRAFと比較してスパイク検出精度の向上
・バブル内参照によるノイズ低減（無関係ユーザの除外）
・HCPでの事前学習により，少ないデータでも機能する予測基盤の構築
・対照学習により h 空間がバブル構造に向けて能動的に整形される
  → split/merge の判定精度向上，バブル構成の安定化
・λ = 0 のベースラインとの比較実験により，対照学習の寄与を定量評価できる
