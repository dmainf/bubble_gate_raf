研究計画：バブル型クラスタ連合学習によるGateRAF強化と脳fMRIタスク予測

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
0. なぜGateRAFか
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

タスク実行中の脳fMRI時系列データは以下の3つの性質を持ち，通常の予測モデルでは対処が難しい．

【系列の多さ】
  - fMRIデータはROI×被験者の組み合わせで膨大な数の時系列が存在する
  - 単一被験者・単一ROIのデータ量は限られていても，系列の総数は多い
  - → 他の被験者・ROIの系列を参照することで情報不足を補える
  - RAFはこの「横断的な参照」を可能にするアーキテクチャである

【スパイク】
  - タスク中の神経応答はBOLD信号として現れ，刺激提示タイミングに同期した
    スパイク状の一過性の活動が平常期の中に点在する
  - 予測モデルは誤差全体を最小化しようとするため，頻度の低いスパイクを無視しがち
  - スパイクそのものを予測するよりも，スパイク直前の予兆（トリガー）を
    他被験者の類似タスク事例から参照することで補完できる
  - → RAFの参照機構がスパイク予測に有効

【スパース】
  - タスク中に活動する脳領域は全ROIのごく一部であり，時系列はスパース
  - スパースな系列では時系列モデルが学習できる特徴量が少ない
  - → 似た脳ネットワーク特性を持つ被験者の隠れ状態を参照して補完できる

【シンプルなRAFでは不十分な理由】
  - 似ていない系列を強制的に参照してしまう → ノイズになる
  - 異なる被験者の時系列を結合すると時系列が不連続になる
  - → Gateで不適切な参照を棄却し，Multi-Streamで系列を分離するGateRAFが必要

【脳データ固有の追加課題】
  - fMRI時系列は個人差が激しく，同じタスクでも被験者間で活動パターンが異なる
  - 個人差によってベクトル空間の基準がずれ，参照がさらに機能しにくい
  - → バブル型クラスタ連合学習で空間を整地することが不可欠

1. 研究の核心
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

バブル型クラスタ連合学習（Clustered Federated Learning: CFL）によって
GateRAFが有効に機能する条件を整え，脳fMRIデータからタスク行動を予測する．

GateRAFが機能するための前提条件：
  - 参照先の隠れ状態が同じベクトル空間の基準を持つこと
  - 似たユーザの隠れ状態が近傍に存在すること

バブルの役割：
  - 個人差・環境差によるベクトル空間のずれを dist_match 層で整地
  - 似た脳ネットワーク特性を持つユーザを同一バブルに集約
  - バブル内でのみRAFを実施 → 意味のある参照が可能になる

バブル管理の核心（改訂設計）：
  - クライアントが「自分の所属バブルのモデル」を通して得た中間表現 h の
    統計量（平均 μ・標準偏差 σ）をサーバーへ送信する
  - サーバーは各クライアントの統計量から分布間距離を計算してバブルを更新する
  - → 「勾配の向きによる間接推定」ではなく「中間表現の分布による直接比較」

2. アーキテクチャ設計：1ラウンドの詳細フロー
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【クライアント側の処理（1ラウンド）】

  ① サーバーから「自分の所属バブルの最新モデル」を受け取る
       (dist_match + head の重み)

  ② 受け取ったモデルでローカル学習（n エポック，Adam，MSELoss）
       入力: z ∈ R^{288}  (SwiFT 凍結エンコーダの出力)
       出力: [acc_2bk_pred, rt_2bk_pred]
       損失: MSE(pred, label)

  ③ 学習後のモデルで全データを forward pass し，dist_match 層の出力
       h = dist_match(z) ∈ R^{128} を取得する

  ④ バッチ全体にわたる h の統計量を計算する
       μ_i ∈ R^{128}  : 次元ごとの平均
       σ_i ∈ R^{128}  : 次元ごとの標準偏差

  ⑤ サーバーへ送信するもの
       - モデル更新量 Δw（重みの差分）
       - 中間表現統計量 {μ_i, σ_i}
       ※ 生データ・潜在ベクトル z は送信しない

【サーバー側の処理（1ラウンド）】

  ① 全クライアントから Δw と {μ_i, σ_i} を受け取る

  ② バブル内 FedAvg：バブルごとに Δw の平均を集約してモデルを更新

  ③ バブル更新（分割・結合）：後述のスケジュールに従って実行

  ④ 更新後のバブル所属情報と，各バブルの最新モデルをクライアントへ返す

3. バブル更新：分布距離に基づく分割・結合
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【分布間距離の計算】

  クライアント i, j の間の分布距離として，ガウス分布を仮定した
  2-Wasserstein 距離の近似を使用する：

    W2²(i, j) = ‖μ_i − μ_j‖² + ‖σ_i − σ_j‖²

  - 計算コストが低く（O(D)，D=128）かつ解釈しやすい
  - 同一バブルのモデルを通した h 同士を比較するため，
    モデルが最適化されるにつれて距離の意味が精緻化される

【分割（Split）：5ラウンドごと】

  バブル内の全ペアについて W2² を計算し，最大距離ペアが閾値を超えたら
  そのペアを軸に 2 グループに分割する（2分割法）：
    - 距離が最大のクライアント i*, j* を見つける
    - 各クライアント k を，i* と j* のどちらに近いかで割り振る

【結合（Merge）：10ラウンドごと】

  バブルごとの代表点（μ̄, σ̄）を計算し，バブル間の W2² が閾値を下回ったら
  2バブルを結合する：
    μ̄_A = mean({μ_i | i ∈ bubble A})
    σ̄_A = mean({σ_i | i ∈ bubble A})
    W2²(A, B) = ‖μ̄_A − μ̄_B‖² + ‖σ̄_A − σ̄_B‖²

4. CFLと分割学習の比較
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ┌───────────────────────┬──────────────────────────┬────────────────────────────────┐
  │                       │   分割学習                 │   CFL（本提案）                 │
  ├───────────────────────┼──────────────────────────┼────────────────────────────────┤
  │ サーバーへの送信物      │  潜在ベクトル z            │  Δw ＋ 統計量 {μ, σ}           │
  │ 個人情報の漏洩リスク    │  中（逆変換の可能性あり）   │  低（統計量からの復元は困難）    │
  │ バブル管理の根拠        │  ベクトル距離              │  中間表現の分布間距離（W2²）     │
  │ 空間整地の場所          │  サーバー側                │  クライアント側（バブルごと）    │
  │ 比較対象の性質          │  SwiFT 直後の生 z          │  タスク適合後の dist_match 出力  │
  └───────────────────────┴──────────────────────────┴────────────────────────────────┘

  優位点：
  - 生データも潜在ベクトルも外部に出ない → 完全なプライバシー保護
  - バブルごとのモデルを通した h を比較するため，
    「このバブルで見たときに似ているか」という文脈依存の距離を実現できる
  - 差分プライバシー（μ, σ へのノイズ付加）と組み合わせることでさらに強化できる

5. バブル形成の3段階パイプライン（HCPを使った事前学習）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 1：初期バブル形成（HCP WorkingMemoryタスクfMRI）

  目標：被験者の生理的・認知的特性に基づく初期クラスタを形成する

  クライアント側：
    HCP WMタスクfMRI（生ボクセル，91×109×91×T）
        ↓
    前処理（96³クロップ・z正規化）
        ↓
    事前学習済みエンコーダ SwiFT（凍結） → z ∈ R^{288}
        ↓
    dist_match 層（学習） → h ∈ R^{128}
        ↓
    予測ヘッド（学習） → [acc_2bk_pred, rt_2bk_pred]
        ↓
    Δw と統計量 {μ, σ} を計算 → サーバーへ送信

  サーバー側：
    バブル内 FedAvg でモデルを更新
    5ラウンドごとに W2² に基づくバブル分割
    10ラウンドごとに W2² に基づくバブル結合
    更新バブルとモデルをクライアントへ返す

【バブル形成の3段階】
  1. 初期：生理的マッチング — SwiFT 直後の z の分布距離で初期バブルを形成
  2. 中期：タスク指向マッチング — dist_match 出力 h の W2² でバブルを動的に再配置
  3. 終期：性能最適化 — モデル収束後の分布安定性でバブル構成を固定

6. 使用データセット
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

HCP (Human Connectome Project) Young Adult
  - 被験者数：1,200名（健常若年成人，22-35歳）
  - タスクfMRI：WorkingMemoryタスク（N-back，LR/RL 2方向）を使用
  - fMRI仕様：2.0mm isotropic，TR=720ms，全脳72スライス，405時点
  - 行動データ：条件別反応時間・正答率が被験者ごとに取得済み
  - アクセス：AWS S3（s3://hcp-openaccess/HCP_1200/）

事前学習済みエンコーダ：SwiFT（NeurIPS 2023）
  - 4D volumetric fMRIを入力とするSwin Transformer
  - HCP・ABCD・UKBiobankで学習済み
  - 入力：(B, 1, 96, 96, 96, 20)，出力：(B, 288, 2, 2, 2, 20) → poolingで (B, 288)
  - 凍結して使用し，dist_match 層と予測ヘッドのみ学習

7. 期待される効果
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

・バブルなしのGateRAFと比較してスパイク検出精度の向上
・バブル内参照によるノイズ低減（無関係ユーザの除外）
・HCPでの事前学習により，少ないデータでも機能する予測基盤の構築
・中間表現統計量のみの通信による完全なプライバシー保護（分割学習との差別化）
・バブルごとのモデルを通した文脈依存距離計算による，より精緻なクラスタリング
